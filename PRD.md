## 1. 产品概述

### 1.1 产品定位

**AI咨询全球**是一个基于 Web3 + AI 的全球化咨询平台，核心理念 **"Code is Law"**，通过智能合约与 AI 自动化实现咨询行业的完全去中心化。

**平台标语**：中国AI算力 · 服务全球 · 收益回流 · 普惠民生

### 1.2 第一阶段目标

| 目标 | 说明 |
|---|---|
| **官网** | 品牌门户，展示平台价值、白皮书、里程碑、社交链接 |
| **NFT铸造** | 开放 500 枚创始股东 NFT 的链上铸造 |
| **Dashboard** | 钱包登录后的个人面板，展示 NFT 资产、权益、推荐链接 |

### 1.3 核心假设

1. 用户已具备 Web3 钱包（MetaMask / WalletConnect 等）
2. 用户持有目标链上的 USDC 用于铸造支付
3. 新加坡公司主体已注册完成
4. 智能合约已完成审计并部署至主网
5. 第一阶段仅开放「合伙人」身份铸造，其余身份（用户/专家/KOL）留待后续

### 3.2 NFT 铸造功能 (P0)

#### 3.2.1 铸造页面布局

| 需求项 | 说明 |
|---|---|
| **页面标题** | "创始股东 NFT 铸造" |
| **核心信息区** | NFT 名称、铸造价格（10,000 USDC）、总量（500 枚）、合约地址 |
| **进度展示区** | 实时数据看板：每条链已铸造数 + 总铸造数 |
| **链选择区** | 4 链选择卡片：Base / BSC / ETH / Solana |
| **铸造按钮** | "铸造合伙人NFT"，未连接钱包时置灰 |

#### 3.2.2 铸造流程

```
[连接钱包] → [选择区块链网络] → [确认铸造信息]
                                        ↓
                              显示：链名称、费用 10,000 USDC
                              显示：Gas 预估
                              显示：铸造后 NFT 等级
                                        ↓
                            [确认支付] → 调用智能合约
                                        ↓
                      [交易确认中...] → 链上确认
                                        ↓
                            [铸造成功] → 展示 NFT 信息
                                        ↓
                              引导进入 Dashboard
```

#### 3.2.3 铸造数据实时展示

| 数据项 | 数据来源 | 更新频率 |
|---|---|---|
| Base 链已铸造数 | 合约查询 | 每 30 秒 |
| BSC 链已铸造数 | 合约查询 | 每 30 秒 |
| ETH 链已铸造数 | 合约查询 | 每 30 秒 |
| Solana 链已铸造数 | 合约查询 | 每 30 秒 |
| 总铸造数 | 合约查询 / The Graph | 每 30 秒 |
| 剩余可铸造数 | 500 - 总铸造数 | 每 30 秒 |

#### 3.2.4 链选择 UI

| 需求项 | 说明 |

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标值 |
|---|---|
| 首屏加载（LCP） | < 2.5 秒（全球 CDN） |
| 交互响应（FID） | < 100ms |
| 累计布局偏移（CLS） | < 0.1 |
| 页面切换时间 | < 500ms |
| 铸造数据刷新 | ≤ 30 秒轮询 → 未来升级为 WebSocket 推送 |
| 白皮书页面渲染 | < 1 秒 |

### 4.2 兼容性

| 维度 | 覆盖范围 |
|---|---|
| **桌面浏览器** | Chrome 90+、Firefox 90+、Safari 15+、Edge 90+ |
| **移动浏览器** | iOS Safari 15+、Android Chrome 90+ |
| **屏幕宽度** | ≥ 320px（移动端） → ≥ 1920px（桌面端） |
| **暗色模式** | 默认强制暗色主题，暂不支持亮色切换 |

### 4.3 可访问性

| 需求项 | 说明 |
|---|---|
| **语义化 HTML** | 正确使用 heading 层级、ARIA 标签 |
| **键盘导航** | Tab 键可导航所有可交互元素 |
| **色觉友好** | NFT 等级不纯靠颜色区分（加图标 + 文字） |
| **图片 alt** | 所有图片提供描述性 alt 文本 |

### 4.4 安全性

| 需求项 | 说明 |
|---|---|
| **HTTPS** | 强制 HTTPS，HSTS 头部 |
| **CSP** | 内容安全策略限制脚本来源 |
| **XSS 防护** | 所有用户输入/URL 参数做转义处理 |
| **钱包连接** | 使用 Web3Modal v3，不存储私钥 |
| **合约交互** | 仅在用户显式确认后发送交易 |

### 4.5 SEO / 社交媒体

| 需求项 | 说明 |
|---|---|
| **Meta 标签** | title、description、og:image、og:title、twitter:card |
| **结构化数据** | JSON-LD Organization schema |
| **多语言 SEO** | hreflang 标签 + 各语言独立 meta |
| **Sitemap** | 自动生成 sitemap.xml |
| **OG 图片** | 统一社交分享图（1200×630） |

## 5. 技术约束与架构边界

### 5.1 技术栈（Phase 1）

| 层级 | 技术选型 | 备注 |
|---|---|---|
| **前端框架** | React 18 + Next.js 14 (App Router) | SSR/SSG 混合渲染 |
| **样式方案** | Tailwind CSS + Framer Motion | 响应式 + 动效 |
| **多语言** | next-intl 或 react-i18next | 9 语言 |
| **钱包连接** | Web3Modal v3 (AppKit) + Wagmi v2 | 多链支持 |
| **合约交互** | Viem + Wagmi | EVM 链 |
| **Solana 支持** | @solana/wallet-adapter + @solana/web3.js | Solana 链 |
| **数据索引** | The Graph (子图) | 铸造数据查询 |
| **CDN** | Cloudflare | 全球加速 |
| **部署** | Vercel / Cloudflare Pages | 边缘部署 |

### 5.2 数据流

```
用户浏览器 ──→ Web3Modal ──→ 钱包签名/交易
      │
      ├──→ Next.js SSR ──→ 静态页面渲染
      │
      ├──→ Wagmi/Viem ──→ RPC 节点 ──→ 链上数据
      │         │
      │         └──→ The Graph 子图 ──→ 铸造统计数据
      │
      └──→ i18n 路由 ──→ 9 语言内容切换
```

### 5.3 边界限制

| 范围 | Phase 1 包含 | Phase 1 不包含 |
|---|---|---|
| **后端 API** | 无自定义后端 | 用户系统、数据库 |
| **身份验证** | 仅钱包签名验证 | 传统用户名/密码 |
| **数据持久化** | 仅链上 + 浏览器本地 | 服务端数据库 |
| **AI 功能** | 无 | AI 客服、智能匹配 |
| **合约部署** | 使用已部署合约 | 合约开发/审计 |
| **链上操作** | 仅铸造/查询 | 治理投票、收益领取 |

## 7. 数据模型概要

### 7.1 前端状态管理

```
AppStore
├── wallet
│   ├── isConnected: boolean
│   ├── address: string
│   ├── chainId: number
│   └── balance: string
├── mint
│   ├── selectedChain: 'base' | 'bsc' | 'eth' | 'solana'
│   ├── chainStats: Record<ChainId, { minted: number; total: number }>
│   ├── totalMinted: number
│   ├── totalSupply: number (500)
│   └── mintStatus: 'idle' | 'confirming' | 'pending' | 'success' | 'error'
├── dashboard
│   ├── nftInfo: { tokenId, chain, contractAddress, mintedAt, level }
│   └── referralCode: string
└── i18n
    ├── locale: string
    └── messages: Record<string, string>
```

### 7.2 智能合约接口（只读查询）

| 合约 | 方法 | 用途 |
|---|---|---|
| NFT 合约 | `totalSupply()` | 获取总铸造数 |
| NFT 合约 | `mintedPerChain(uint256 chainId)` | 获取某链铸造数 |
| NFT 合约 | `mint(uint256 chainId)` | 铸造 NFT |
| NFT 合约 | `tokenOfOwnerByIndex(address, uint256)` | 按索引查用户 Token |
| NFT 合约 | `getTokenLevel(uint256 tokenId)` | 获取 NFT 等级 |

> 具体合约 ABI 和地址由智能合约开发团队交付后集成。

### 9.2 翻译范围

- **P0 必须翻译**：繁体中文、英文（覆盖全站）
- **P1 后续翻译**：其余 7 语言
- **不翻译项**：合约地址、Token ID、代码片段、区块链术语（保持英文）
- **文案管理**：JSON/ICU MessageFormat 格式，按页面/组件拆分

### 9.3 RTL 支持

阿拉伯语 (`ar`) 需要 RTL 布局：
- 使用 CSS 逻辑属性（`margin-inline-start` 替代 `margin-left`）
- Tailwind RTL 插件或手动适配
