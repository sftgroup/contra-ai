## 2. 零撸释放算法

### 2.1 核心设计原则

- **总量硬顶**：37.5 亿 $CAI，耗尽即停止，无增发
- **递减释放**：随时间/里程碑逐步降低释放速率，避免早期过度稀释
- **先到先得 + 后期可持续**：早期优势存在但有限，长期持有者也受益
- **防羊毛**：每日上限 + KYC/行为验证联动

### 2.2 每日总释放上限公式

设每日总释放上限为 D_n，其中 n 为自 App 上线起的天数：

```
D_n = D_0 × 1/(1 + k·n)^s × M
```

**参数说明：**

| 参数 | 值 | 说明 |
|------|-----|------|
| D_0 | 5,000,000 $CAI | 初始每日总上限 |
| k | 0.001 | 衰减系数 |
| s | 0.5 | 衰减指数（平方根衰减，温和递减） |
| M | — | 里程碑复合乘数，见 §5 |

**里程碑乘数 M 默认 = 1.0，触发后调整**：

```
M = M_user × M_expert × M_node × M_tech
```

**硬约束**：累计释放量 ≤ 37.5 亿 $CAI。当累计释放达到上限时，D_n = 0。

### 2.3 单用户每日领取公式

```
U_n = min( D_n × 1/N_active ,  U_base × B_n × M )
```

| 参数 | 初始值 | 说明 |
|------|--------|------|
| N_active | — | 当日活跃零撸用户数 |
| U_base | 500 $CAI | 单个用户基础日领上限 |
| B_n | — | 行为系数 0.3-1.5，基于签到/任务/社交验证 |

**U_base 随时间衰减**：

```
U_base(n) = 500 × 1/(1 + 0.0008·n)^0.5
```

**行为系数 B_n**：

| 行为 | 加分 |
|------|------|
| 基础签到 | 0.3 |
| 完成 KYC 验证 | +0.3 |
| 完成每日任务 | +0.2 |
| 绑定社交账号 | +0.3 |
| 链上交互（gas消费验证） | +0.2 |
| 连续签到加成（第 7 天 ×1.2） | 乘数 |

```
B_n = min(1.5, 0.3 + kycB + taskB + socialB + chainB)
```

### 2.4 邀请奖励机制

**规则**：一级邀请，邀请人获得被邀请人当日零撸量的 **10%**，从平台零撸总池额外支出（**不扣被邀请人**）。

**每日邀请奖励上限**：

单个用户的邀请奖励日上限 = `U_base(n) × 10 × M`

（即最多相当于 10 个活跃被邀请人的收益）

**硬约束**：邀请总奖励 + 零撸直接领取 ≤ 37.5 亿。

```
C_claim + C_invite ≤ 3,750,000,000
```

**调整策略**：

| 累计消耗占比 | 邀请奖励比例 |
|------------|------------|
| 0% - 80% | 10% |
| 80% - 95% | 5% |

## 7. 专家铸造 → 挂单买入 → 价格曲线模拟

### 7.1 挂单价格形成机制

**前提假设**：
- App 上线时，用户通过零撸获得代币
- 用户可自行选择挂单价格（自由挂单）
- 合约按价格从低到高买入

**合约买入规则**：

```
FOR each 专家铸造:
  1. 专家存入 10,000 USDC
  2. 合约取出 5,000 USDC
  3. 按挂单价格从低到高依次买入
  4. 直到 5,000 USDC 完全花完或挂单清空
  5. 买入的代币转入该专家的 NFT 合约关联钱包

  风控：单笔挂单上限 = 5,000 USDC 等值
       挂单最低价格 = 0.0001 USDC（防止无意义低价刷榜）
```

### 7.2 用户挂单行为假设

用户在零撸中获得代币后：

| 用户类型 | 占比 | 挂单行为 |
|---------|------|---------|
| 短期套利者 | ~30% | 低价挂单（$0.0001 - $0.001），快速变现 |
| 观望持有者 | ~50% | 不挂单，等待流动性池建立后交易 |
| 高价试探者 | ~20% | 较高价挂单（$0.005 - $0.01），测试市场心理价位 |

### 7.3 100 个专家铸造成交价格阶梯

**最终价格**：第 100 个专家铸完后，市场最后一笔成交价约 **$0.009 - $0.01 USDC** ✅

### 7.4 流动性池建立后初始价格

**池子组成**：
- USDC 侧：**50 万**（来自 100 个专家的 5,000 USDC × 100）
- 代币侧：等值代币 = 50 万 ÷ 最新成交价

设最后成交价为 $0.0096 USDC：

```
代币数量_pool = 500,000 ÷ 0.0096 ≈ 52,083,333 CAI
```

**初始恒定乘积**：

```
k_0 = 500,000 × 52,083,333 = 2.604 × 10^13
```

**初始现货价格**：

```
P_0 = USDC ÷ CAI = 500,000 ÷ 52,083,333 = 0.0096 USDC/CAI
```

✅ 落在目标区间 **$0.001 - $0.01 USDC** 的上沿附近

## 8. 砸盘风险分析

### 8.1 用户持币行为建模

| 用户类型 | 占比 | 平均持仓 | 卖出倾向 | 卖出时机 |
|---------|------|---------|---------|---------|
| 纯撸毛党 | 25% | 500-2,000 CAI | 极高 | 有价格立即卖 |
| 投机散户 | 35% | 2,000-10,000 CAI | 中等 | 涨 50% 卖一半 |
| 长期信仰者 | 25% | 5,000-50,000 CAI | 低 | 涨 5-10x 才分批 |
| 大户/节点 | 10% | 50,000+ CAI | 极低 | 生态利益绑定 |
| 专家 | 5% | 极高（含铸造购入） | 极低 | 平台运营需求 |

### 8.2 流动性池建立时的卖压计算

**潜在卖压**（流动性池建立后，用户开始通过 DEX 卖出）：

| 场景 | 假设条件 | 潜在卖压 | 最大价格冲击 |
|------|---------|---------|------------|
| 保守 | 25%纯撸 + 10%投机卖出 | ~1,000 万 CAI | **-15%** |
| 中性 | 25%纯撸 + 25%投机卖出 | ~1,800 万 CAI | **-28%** |
| 悲观 | 25%纯撸 + 35%投机 + 5%信仰者 | ~2,800 万 CAI | **-45%** |

**中性场景计算**（恒定乘积 AMM）：

卖出 1,800 万 CAI 入池：

```
P_after = (500,000 - ΔUSDC) / (52,083,333 + 18,000,000)

ΔUSDC = 500,000 - (500,000 × 52,083,333) / 70,083,333 ≈ 128,400 USDC

P_after = 371,600 / 70,083,333 ≈ 0.0053 USDC
```

价格下跌约 **-45%**（从 0.0096 → 0.0053）

### 8.3 风险缓解措施

| 措施 | 机制 | 预期效果 |
|------|------|---------|
| **解锁分批** | 流动性池上线后 7 天才允许提币（7 天冷静期） | 消化恐慌情绪 |
| **卖出税** | 前 30 天卖出收 5% 税，逐步降至 0% | 抑制短期抛售 |
| **挂单锁仓奖励** | 挂单不取消的用户获得额外零撸加成 | 减少撤回挂单行为 |
| **Staking 即开** | 流动性池上线同期开启 staking（APR 12.8%） | 锁定流通量 |
| **做市商激励** | 提供 LP 的代币持有者获额外 $CAI 奖励 | 增加流动性深度 |
| **专家持续买入** | 100 个专家之后，新增专家铸造继续用 5000U 从 DEX 买入 | 持续需求支撑 |

**综合效果**：中性场景下实际价格冲击预计 **-15% 到 -25%**（考虑 staking 锁仓 + 卖出税 + 冷静期）
